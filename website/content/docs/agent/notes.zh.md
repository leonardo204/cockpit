Cockpit 的**笔记**是给那些不该进 commit、也不该进聊天的东西用的草稿本 —— 备忘、todo 列表、常看的网页链接、临时计算,任何你想跨会话留住的常用信息。分两种:

| 范围 | 何时用 |
|---|---|
| **全局笔记** | 跨项目的备忘、todo、收藏链接。不管在哪个项目 tab 都看得到。 |
| **项目笔记** | 钉在某个项目上的笔记 —— 决定、路径、临时计算,只在那个代码库里才有意义。 |

两种范围用同一套编辑器、同一套自动保存,只是存的位置不同。

## 打开笔记

- **全局笔记** —— 左侧栏底部"Notes"按钮(在定时任务区域下面,Skills 上面)。Cockpit 整个**只有一份**全局笔记,所有项目共享。
- **项目笔记** —— 左侧栏每个项目项上鼠标悬停,出现的小记号按钮(也可以从 Agent 输入框旁的"项目笔记"图标进入)。**每个项目一份**,按项目目录隔离 —— `~/code/backend` 和 `~/code/frontend` 的笔记是两份独立的。

每种范围都只有一份笔记 —— 没有"新建笔记"或"切换笔记"的概念要操心。

## 干什么用

平时塞进笔记的东西:

- **备忘** —— "明天记得问 AI 关于 X"、"staging 的某变量在哪"、待会儿要回想的决定。
- **Todo 管理** —— `/task` 复选框列表,边干边勾。在你自己脑子里跑的待办,不用进 issue tracker。
- **网页链接** —— 粘一个想再开的 URL:正在读的文档、要 review 的 PR、晚点要修的 issue。
- **代码片段** —— 每周要敲的那条 shell 命令、永远记不住的那个正则。
- **聊天回复抄录** —— 把有用的 AI 回复从聊天里抄出来,这样会话刷新 / 清掉 / 关掉后还能找到。

全局笔记装那些跟着你走的内容;项目笔记装只在一个代码库里有意义的。

## 编辑器

基于 Tiptap 的富文本编辑器,带 Markdown 感:

- 标题(H1 / H2 / H3) —— 打 `# title` 或用斜杠菜单。
- 项目符号列表、编号列表、任务列表(复选框)。
- 引用块、代码块、表格、分割线。
- 链接。
- 内置撤销 / 重做(StarterKit 自带 History)。

### 斜杠菜单

打 `/` 弹出格式化命令菜单 —— **共 11 个内置动作**:

| `/` | 插入 |
|---|---|
| `/h1`、`/h2`、`/h3` | 各级标题 |
| `/bullet` | 项目符号列表 |
| `/ordered` | 编号列表 |
| `/task` | 任务列表(复选框) |
| `/quote` | 引用块 |
| `/code` | 代码块 |
| `/table` | 3×3 表格 |
| `/hr` | 分割线 |
| `/link` | 插入链接 |

这些**只**在笔记编辑器里生效 —— 聊天输入框里没有。聊天输入有[它自己另一套斜杠菜单](/zh/docs/agent/skills/)给 Skills 用。

## 自动保存

两种笔记都在你停止打字大约 5 秒后自动保存。**没有"保存"按钮**;编辑过程中右上角会显示一个 "saving..." 小指示器。编辑到一半关掉模态,待保存的改动会先 flush 落盘。

## 限制

- **不能嵌入图片** —— 只支持文本和代码。
- **没有文件附件。**
- **没有跨 tab 实时同步** —— 同一份笔记在两个窗口打开,后保存的覆盖先保存的。
- 笔记**不分享**给协作者 —— 是你机器本地的。要分享的文档用[技术方案评审](/zh/docs/explorer/reviews/#创建与分享)。
- **没有跨笔记搜索** —— 笔记编辑器没有内置 search,要找东西用浏览器的 `Cmd+F`。
