**最近** 标签是 Explorer 面板的 "我刚才看的是什么" 视图。文件按**最后访问时间**排序,最新的在最上面。

## 什么算 "访问过"

凡是在 Cockpit 里实际打开过的文件:

- 在文件树里点开。
- 从 `Cmd+P`（快速打开文件）跳过去。
- 通过 `Cmd+点击` 跳到定义后落到的文件。
- 在 Git **历史** 标签的提交详情面板或 diff 查看器里点开的文件。

只是在目录列表里**看到**的不算 —— 必须真的打开过。

## 不止是路径 —— 光标和滚动位置也保留

每次你在某文件里滚动或移动光标,Cockpit 把那个文件的:

- **当前滚动行**
- **光标行**
- **光标列**

一起记进最近列表。下次从最近里点回来,Cockpit 把视图滚回原位、光标摆回原处 —— 不用再翻一遍找你刚才看到哪。

## 上限和淘汰

每个项目保留 **最近 15 个** 文件。打开第 16 个文件时,最旧的从底部掉出。最近列表按项目隔离 —— 项目 A 的最近不会出现在项目 B。

列表持久化在你机器上,Cockpit 重启后还在。

## 没有的功能

为了避免误解,**这些不存在**:

- 没有 **清空最近** 按钮。最近就是个滚动的访问日志,凑齐 15 条之后旧的自动掉。
- 没有 **逐条删除**。
- 没有 **钉到顶部** / pin 收藏。要长期摆在显眼位置的文件,在文件树里手动展开它的目录路径就好。

## 跟其它 tab 的关系

|  | 它干什么 |
|---|---|
| **目录树** | 看项目所有文件 —— 完整结构。 |
| **搜索** | 按文件名或内容找,用得多 → 自动进最近。 |
| **最近** | 已经打开过的子集,光标位置自动跟随。 |
